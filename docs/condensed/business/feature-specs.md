# 📋 功能規格

## 核心功能模組

### 1. 輸入模式

#### A. 手動輸入模式 (首頁)
**功能**: 單產品快速分析
- **輸入欄位**: 產品名稱、當前價格、成本、流量、運費、交易費率
- **即時分析**: 輸入變更即時重新計算
- **視覺化**: 互動式滑桿和圖表
- **適用場景**: 快速測試、概念驗證

#### B. CSV批量模式 (分析頁面)
**功能**: 多產品批量分析
- **支援格式**: CSV檔案 (Shopify導出格式)
- **產品篩選**: 按類別、供應商、狀態篩選
- **搜尋功能**: 產品名稱快速搜尋
- **分頁顯示**: 預設顯示39個產品 (可擴展)

### 2. 優化評估準則 (OECs)

| OEC選項 | 業務目標 | 計算邏輯 | 使用時機 |
|---------|----------|----------|----------|
| **最大化營收** | 提升總銷售額 | `Price × Traffic × ConversionRate` | 市場擴張期、新產品推廣 |
| **最大化利潤** | 提升獲利能力 | `(Price - Cost) × Traffic × ConversionRate` | 成熟產品、利潤優化 |
| **最大化轉換率** | 提升客戶獲取 | `max(ConversionRate)` | 客戶基礎建立、競爭對抗 |

### 3. 互動式價格分析

#### A. 雙滑桿控制系統
- **原價滑桿**: 固定參考價格 (藍色標記)
- **測試價格滑桿**: 可調整測試價格 (綠色標記)  
- **最佳價格標記**: AI計算最佳點 (紅色標記)
- **快速設定**: "Set to Optimal" 一鍵跳轉

#### B. 即時計算反饋
- **計算週期**: 100ms防抖優化
- **顯示指標**: 轉換率%、營收$、利潤$
- **變化幅度**: 百分比和絕對值變化
- **風險指示**: 顏色編碼風險等級

### 4. 數據視覺化

#### A. 比較長條圖
```typescript
數據維度: [轉換率%, 利潤$, 營收$]
比較對象: [原始價格, 新價格, 最佳價格]
顏色編碼: [藍色, 綠色, 紅色]
```

#### B. 價格範圍視覺化
- **背景條**: 價格區間視覺化
- **梯度效果**: 價格影響程度漸變
- **標記系統**: 關鍵價格點標記

## 技術規格

### 1. 資料處理

#### CSV檔案支援
```typescript
interface ProductVariant {
  handle: string;              // 產品代碼
  title: string;               // 產品名稱
  variantOption: string;       // 規格選項
  price: number;               // 當前價格
  compareAtPrice?: number;     // 比較價格
  costPerItem: number;         // 單位成本
  requiresShipping: boolean;   // 是否需運費
  vendor: string;              // 供應商
  category: string;            // 產品類別
  sku?: string;               // SKU代碼
  weight?: number;            // 重量
  tags?: string;              // 標籤
}
```

#### 數據驗證規則
- **必要欄位**: title, price, costPerItem
- **數值範圍**: price > 0, costPerItem ≥ 0
- **邏輯驗證**: price > costPerItem (合理利潤率)

### 2. 計算引擎

#### 性能規格
- **即時計算**: <100ms 回應時間
- **批量處理**: 支援1000+產品同時分析
- **記憶體使用**: <50MB (大型CSV檔案)
- **準確度**: 數值計算精確到小數點後2位

#### 容錯處理
- **無效數據**: 自動跳過並警告
- **極值處理**: 自動限制在合理範圍內
- **錯誤恢復**: 局部錯誤不影響整體分析

### 3. 用戶體驗規格

#### 回應性設計
- **手機端**: 375px+ 完整功能
- **平板端**: 768px+ 優化佈局
- **桌面端**: 1440px+ 最佳體驗

#### 互動回饋
- **載入狀態**: 骨架屏 + 進度指示
- **錯誤提示**: Toast通知 + 內聯錯誤
- **成功確認**: 視覺化完成提示

## 業務邏輯規則

### 1. 定價建議邏輯

#### 合理性檢查
```typescript
// 價格合理性
const margin = (price - cost) / price;
if (margin < 0.1) warning("利潤率過低");
if (margin > 0.8) warning("定價可能過高");

// 價格變動幅度
const changePercent = Math.abs((newPrice - oldPrice) / oldPrice);
if (changePercent > 0.3) warning("價格變動過大，建議分階段調整");
```

### 2. 風險評估等級

| 風險等級 | 條件 | 建議動作 |
|----------|------|----------|
| **低風險** | 價格變動 < 10% | 可直接執行A/B測試 |
| **中風險** | 價格變動 10-20% | 建議小規模測試 |
| **高風險** | 價格變動 > 20% | 需要謹慎評估市場反應 |

### 3. 商業場景對應

#### 電商常見場景
1. **新品定價**: 使用轉換率最大化策略建立市場
2. **促銷優化**: 營收最大化找出最佳折扣點
3. **利潤提升**: 利潤最大化優化成熟產品定價
4. **清庫存**: 轉換率優化快速出清存貨

#### 行業適用性
- ✅ **電商零售**: 完全適用
- ✅ **SaaS服務**: 適用於訂閱定價
- ✅ **實體零售**: 需調整流量計算邏輯
- ⚠️ **B2B銷售**: 需考慮銷售週期差異
